# Impeccable Design Guidelines

## Design Context

### Users
- **谁**：游戏玩家（年轻男性为主），边打游戏边使用
- **场景**：深夜连麦、游戏间隙闲聊、对战中实时交流
- **任务**：与 AI 角色进行轻松有趣的语音/文字互动，获得陪伴感和娱乐感
- **设备**：PC 桌面端浏览器，通常全屏或半屏使用（旁边是游戏）
- **产品定位**：START 云游戏平台的配套工具

### Brand Personality
- **三个关键词**：克制、沉稳、工具感
- **语调**：不花哨、不卖萌，像一个可靠的伙伴在旁边
- **情感目标**：让用户觉得"有个人在陪我打游戏"，同时视觉上融入 START 生态
- **品牌对齐**：作为 START 平台的功能扩展，视觉语言必须和 START 一致

### Aesthetic Direction
- **视觉基调**：深黑底色 + 科技蓝强调，极简克制，大面积留黑
- **参考**：START 云游戏平台（纯黑背景、蓝色强调、细边框、小字号）
- **反参考**：
  - 不要 AI slop（紫色发光、pulse 动画、glassmorphism）
  - 不要企业 SaaS（浅色、大量留白、正经感）
  - 不要过度装饰（gradient text、emoji、bouncy 动画）
- **主题**：纯暗色，蓝色只用在关键交互点（active tab、按钮、focus ring）

### Color Palette
- **背景层级**：
  - 最深底色：`#0e0e0e`
  - 面板背景：`#161616`
  - 卡片/输入框：`#1e1e1e`
  - 悬浮/高亮：`#262626`
- **强调色（START 蓝）**：
  - 主色：`#3b9eff`
  - 主色浅：`#64b5ff`
  - 主色暗：`#1a7de0`
  - 微妙底色：`rgba(59, 158, 255, 0.08)`
- **语义色**：
  - 成功/在线：`#4caf50`
  - 警告：`#f5a623`
  - 错误：`#e04848`
- **文字（冷白系）**：
  - 主文字：`#eaeaea`
  - 次要文字：`#8c8c8c`
  - 弱化文字：`#555555`
- **边框**：`#222222` / `#303030`（极低对比，不抢注意力）

### Typography
- **主字体**：`"PingFang SC", "Microsoft YaHei", -apple-system, sans-serif`（中文优先）
- **等宽字体**：`"JetBrains Mono", "Fira Code", "Consolas", monospace`
- **字号层级**：
  - 应用标题：14px / medium
  - 正文/聊天：13px / regular
  - 辅助/标签：12px / regular
  - 状态/提示：11px
- **行高**：聊天消息 1.6，转写文本 1.7

### Spacing & Layout
- **基础单位**：4px
- **常用间距**：4, 8, 10, 12, 16, 20, 24px
- **圆角**：3px (xs), 5px (sm), 8px (md), 12px (lg)
- **布局**：左右 3:2 分栏（屏幕监控为主），顶栏 48px
- **面板最小宽度**：右侧对话面板 min-width 340px

### Interaction States
- **按钮 hover**：背景变为 `--bg-hover` 或 `--accent-subtle`
- **Focus ring**：`box-shadow: 0 0 0 2px rgba(59, 158, 255, 0.08)` + border 变蓝
- **Active/按下**：实色背景 `--accent` + 白色文字
- **Disabled**：opacity 0.5 + cursor 变化
- **Connecting**：opacity 0.5 + cursor wait

### Design Principles

1. **融入 START** — 视觉语言和 START 平台保持一致，用户不会觉得是第三方插件。
2. **克制表达** — 蓝色只出现在需要注意的地方（active、accent、focus），其余全是黑灰。
3. **即时反馈** — 每个交互都有状态反馈（连接中、聆听中、服务繁忙），且用中文友好文案。
4. **不打断** — 信息提示用 inline toast / statusText，不用 alert() 或 modal 弹窗。
5. **健壮优先** — 所有错误有友好提示，所有按钮防重复点击，所有资源正确清理。
