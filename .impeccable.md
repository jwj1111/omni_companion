# Impeccable Design Guidelines

## Design Context

### Users
- **谁**：游戏玩家（年轻男性为主），边打游戏边使用
- **场景**：深夜连麦、游戏间隙闲聊、对战中实时交流
- **任务**：与 AI 角色进行轻松有趣的语音/文字互动，获得陪伴感和娱乐感
- **设备**：PC 桌面端浏览器，通常全屏或半屏使用（旁边是游戏）

### Brand Personality
- **三个关键词**：游戏感、轻松、有温度
- **语调**：不严肃、不商务，像跟朋友开黑时的氛围
- **情感目标**：让用户觉得"有个人在陪我打游戏"，而不是"在用一个 AI 工具"

### Aesthetic Direction
- **视觉基调**：深色主题，游戏面板感，紧凑但不压抑
- **参考**：Steam 大屏幕模式（大字体、高对比、圆角卡片、深色背景）
- **反参考**：不要像企业 SaaS（浅色、大量留白、正经感）；不要像极简文档工具
- **主题**：暗色为主，搭配高饱和度强调色点缀

### Color Palette
- **背景层级**：
  - 最深底色：`#0d0d0f`（近纯黑，微带冷色）
  - 面板背景：`#1a1b1e`
  - 卡片/输入框：`#25262b`
  - 悬浮/高亮：`#2c2d32`
- **强调色**：
  - 主色：`#7c5cfc`（紫色，活力+游戏感）
  - 主色浅：`#9b7dff`
  - 主色暗：`#5a3fd6`
- **语义色**：
  - 成功/在线：`#4caf50`
  - 警告：`#ff9800`
  - 错误：`#f44336`
  - 信息：`#5c9ce6`
- **文字**：
  - 主文字：`#e4e4e7`
  - 次要文字：`#9ca3af`
  - 弱化文字：`#6b7280`
- **边框**：`#2d2e33`（低对比，不抢注意力）

### Typography
- **主字体**：系统字体栈 `"Inter", -apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif`
- **等宽字体**：`"JetBrains Mono", "Fira Code", monospace`（用于状态/技术信息）
- **字号层级**：
  - 标题：18px / bold
  - 正文/聊天：14px / regular
  - 辅助信息：12px / regular
  - 状态栏：11px
- **行高**：聊天消息 1.5，标题 1.3

### Spacing & Layout
- **基础单位**：4px
- **常用间距**：8px, 12px, 16px, 24px
- **圆角**：
  - 小元素（按钮、标签）：6px
  - 卡片/面板：10px
  - 气泡消息：12px
- **布局**：左右 50/50 分栏，固定顶栏 + 状态栏

### Design Principles

1. **游戏优先** — UI 不能抢游戏的注意力。信息密度高、占屏少、一眼能看完。
2. **暗色护眼** — 深夜使用友好，对比度够但不刺眼，减少大面积亮色。
3. **即时反馈** — 每个交互都有视觉/动效反馈（发送、连接、断开、说话中）。
4. **角色感** — UI 中体现"在跟一个人聊天"的感觉（头像、消息气泡、语音波形）。
5. **不打断** — 信息提示用非侵入方式（toast/微动画），不用弹窗打断用户。
